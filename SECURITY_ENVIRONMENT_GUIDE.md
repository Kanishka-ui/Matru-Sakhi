# 🛡️ MatruSakhi Security Environment Configuration Guide

## 🔐 Environment Security Best Practices

This guide explains how to properly configure environment variables for MatruSakhi to ensure secure deployment and prevent accidental exposure of sensitive credentials.

---

## 📁 Environment File Structure

### 1. Backend (FastAPI)
- `.env` - Local development configuration (NOT committed to Git)
- `.env.example` - Template with placeholders (COMMITTED to Git)
- `.env.production` - Production configuration (NOT committed to Git)

### 2. Frontend (Next.js)
- `.env.local` - Local development configuration (NOT committed to Git)
- `.env.example` - Template with placeholders (COMMITTED to Git)
- `.env.production` - Production configuration (NOT committed to Git)

### 3. AI Backend
- `.env` - Local development configuration (NOT committed to Git)
- `.env.example` - Template with placeholders (COMMITTED to Git)

---

## 🚫 What NOT to Commit to GitHub

**NEVER commit these files to your repository:**
- `backend/.env`
- `client/.env.local`
- `matrusakhi-ai-backend/.env`
- `backend/.env.production`
- `client/.env.production`

These files contain sensitive information like:
- API keys (Groq, OpenAI, Twilio)
- Database credentials
- SMTP passwords
- JWT secrets
- Authentication tokens

---

## ✅ What TO Commit to GitHub

**ALWAYS commit these template files:**
- `backend/.env.example`
- `client/.env.example`
- `matrusakhi-ai-backend/.env.example`

These files contain only placeholders and documentation, showing other developers exactly what environment variables are needed without exposing real credentials.

## 🛠️ Setup Instructions for New Developers

### Step 1: Clone the Repository
```bash
# Clone the repository
git clone https://github.com/your-username/MatruSakhi.git
cd MatruSakhi
```

### Step 2: Create Environment Files
```bash
# Backend
cd backend
cp .env.example .env

# Client
cd ../client
cp .env.example .env.local

# AI Backend
cd ../matrusakhi-ai-backend
cp .env.example .env
```

### Step 3: Configure Your Environment Variables
Edit each `.env` file and replace placeholders with your actual values:

#### Backend `.env`:
```env
SECRET_KEY=your-unique-secret-key-here
OPENAI_API_KEY=sk-your-openai-api-key
GROQ_API_KEY=gsk-your-groq-api-key
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
TWILIO_ACCOUNT_SID=AC-your-twilio-sid
TWILIO_AUTH_TOKEN=your-auth-token
TWILIO_PHONE_NUMBER=+1234567890
```

#### Client `.env.local`:
```env
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXTAUTH_SECRET=your-nextauth-secret
```

#### AI Backend `.env`:
```env
GROQ_API_KEY=your-groq-api-key
OPENAI_API_KEY=your-openai-api-key
SECRET_KEY=your-ai-backend-secret
```

### Step 4: Start Development Servers
```bash
# Terminal 1: Backend
cd backend
python -m uvicorn app.main:app --reload --port 8000

# Terminal 2: AI Backend
cd matrusakhi-ai-backend
python -m uvicorn main:app --reload --port 8001

# Terminal 3: Frontend
cd client
npm run dev
```

## 📝 Environment Variable Reference

| Category | Variable | Purpose | Required |
|----------|----------|---------|----------|
| **Security** | `SECRET_KEY` | JWT signing key | ✅ |
| **AI** | `OPENAI_API_KEY`, `GROQ_API_KEY` | LLM access | ✅ (at least one) |
| **Email** | `SMTP_USER`, `SMTP_PASSWORD` | Email verification | ❌ (optional) |
| **SMS/Voice** | `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_PHONE_NUMBER` | SOS alerts | ❌ (optional) |
| **Database** | `MONGODB_URL` | Database connection | ✅ |
| **Frontend** | `FRONTEND_URL`, `NEXT_PUBLIC_API_URL` | URL configuration | ✅ |

## ⚠️ Critical Security Warnings

### 1. Never Hardcode Secrets
❌ **Bad:** `SECRET_KEY = "my-secret"` in Python code
✅ **Good:** `SECRET_KEY = os.getenv("SECRET_KEY")` from environment

### 2. Use Different Keys for Each Environment
- Development: Use simple keys
- Staging: Use separate keys
- Production: Use strong, unique keys generated by tools like `openssl rand -base64 32`

### 3. Rotate Keys Regularly
- Rotate API keys every 3-6 months
- Immediately rotate if compromised
- Use environment-specific keys

### 4. GitHub Security Scanning
GitHub automatically scans for common secret patterns. If you accidentally commit a secret, GitHub will alert you and provide guidance to remove it.

---

## 🔄 Secret Rotation Procedure

1. Generate new secret: `openssl rand -base64 32`
2. Update `.env` files with new secret
3. Restart all services
4. Test functionality thoroughly
5. Remove old secret from service providers (if applicable)

## 📋 Quick Checklist Before Pushing to GitHub

✅ All `.env` files are in `.gitignore`
✅ Only `.env.example` files are committed
✅ No API keys or passwords in source code
✅ Sensitive data is loaded via `os.getenv()`
✅ Documentation shows proper setup instructions
